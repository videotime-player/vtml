VideoTime Media List (VML1) and VideoTime List Group (VLG1) format specification

Version 3.0.0mirabel

What are lists in VideoTime Media List (VML1) format?

Lists in VideoTime Media List (VML1) format is an XML-based online multimedia list format that allows you to store multimedia content hosted on the network and distribute it to Android devices through the VideoTime client

What are list groups in VideoTime List Group (VML1) format?

Groups of lists in VideoTime List Group (VLG1) format are similar to lists in VideoTime Media List (VML1) format, but unlike lists in VideoTime Media List (VML1) format, groups of lists in VideoTime List Group format (VLG1) allow to store several lists in VideoTime Media List (VML1) format to easily access them, avoiding having to add each of the stored lists to the VideoTime client's list manager

Operation of lists (VML1) and groups of lists (VLG1)

The VideoTime client when loading the list or group of lists from the Internet, connects to the server and downloads the list or group of lists to the client, once downloaded, the data of the list or group of lists is processed and finally the client VideoTime will start to display the items in the list or group of lists

Advantages of lists in VideoTime Media List (VML1) format compared to lists in M3U format

·         Ability to use covers in DVD box size

·         Faster loading, to being based on XML

·         Ability to include a description for each multimedia element

List file structure

File extension: .vml1

Tags:

<vtml-list> List definition

<vtml-mediaitem> Definition of the multimedia element, it is placed inside <vtml-list>

<vtml-itemname> Definition of the element name, it is placed inside <vtml-mediaitem>

<vtml-itemcover> Definition of the cover image URL of the element, it is placed inside <vtml-mediaitem>

<vtml-itemurl> Definition of the multimedia URL of the element, it is placed inside <vtml-mediaitem>

<vtml-itemdescription> Definition of the element's description, it is placed inside <vtml-mediaitem>

<vtml-itemcategory> Definition of the category of the element, it is placed inside <vtml-mediaitem>

<vtml-useragent> (Optional) Definition of the user-agent of the element, it is placed inside <vtml-mediaitem>

<vtml-referer> (Optional) Definition of the referer of the element, it is placed inside <vtml-mediaitem>

<vtml-uselegacystack> (Optional) Defines whether the element should use the Android default network stack instead of the OkHttp network stack in ExoPlayer, it is placed inside <vtml-mediaitem>

<vtml-cctrack> (Optional) Definition of the subtitle file (.SRT) URL of the element, it is placed inside <vtml-mediaitem>

<vtml-cclanguage> (Optional) Definition of the subtitle file (.SRT) language code (ISO 639-2) of the element, it is placed inside <vtml-mediaitem>

<vtml-radiologo> (Opcional) Definition of the radio logo image URL of the element, it is placed inside <vtml-mediaitem>

<vtml-backdrop> (Opcional) Definition of the backdrop image URL of the element, it is placed inside <vtml-mediaitem>

CineInfo List Extension tags:

<cineinfo-year> (Optional) Definition of the year of the element, it is placed inside <vtml-mediaitem>

<cineinfo-producer> (Optional) Definition of the producer of the element, it is placed inside <vtml-mediaitem>

<cineinfo-language> (Optional) Definition of the language of the element, it is placed inside <vtml-mediaitem>

<cineinfo-copyright> (Optional) Definition of the copyright of the element, it is placed inside <vtml-mediaitem>

Code structure:

<?xml version="1.0" encoding="UTF-8"?>

<vtml-list>

       <vtml-mediaitem>

             <vtml-itemname>[NAME]</vtml-itemname>   

             <vtml-itemcover>[COVER]</vtml-itemcover>

             <vtml-itemurl>[URL]</vtml-itemurl>

             <vtml-itemdescription>[DESCRIPTION]</vtml-itemdescription>

             <vtml-itemcategory>[CATEGORY]</vtml-itemcategory>

       </vtml-mediaitem>

</vtml-list>   

Example:

<?xml version="1.0" encoding="UTF-8"?>

<vtml-list>

       <vtml-mediaitem>

             <vtml-itemname>Hello World</vtml-itemname>   

             <vtml-itemcover>http://www.example.com/media/cover.png</vtml-itemcover>

             <vtml-itemurl>http://www.example.com/media/video.mp4</vtml-itemurl>

             <vtml-itemdescription>This is a video sample</vtml-itemdescription>

             <vtml-itemcategory>Samples</vtml-itemcategory>

       </vtml-mediaitem>

       <vtml-mediaitem>

             <vtml-itemname>Hello World 2</vtml-itemname>   

             <vtml-itemcover>http://www.example.com/media/cover.png</vtml-itemcover>

             <vtml-itemurl>http://www.example.com/media/video2.mp4</vtml-itemurl>

             <vtml-itemdescription>This is another video sample</vtml-itemdescription>

             <vtml-itemcategory>Samples</vtml-itemcategory>

       </vtml-mediaitem>

</vtml-list>   

List group file structure

File extension: .vlg1

Tags:

<vtml-group> List group definition

<vtml-listitem> Definition of the list element, it is placed inside <vtml-group>

<vtml-itemname> Definition of the element name, it is placed inside <vtml-listitem>

<vtml-itemcover> Definition of the cover image URL of the element, it is placed inside <vtml-listitem>

<vtml-itemurl> Definition of the element's list URL, it is placed inside <vtml-listitem>

<vtml-itemdescription> Definition of the element description, it is placed inside <vtml-listitem>

<vtml-itemcategory> Definition of the element's category, it is placed inside <vtml-listitem>

Code structure:

<?xml version="1.0" encoding="UTF-8"?>

<vtml-group>

       <vtml-listitem>

             <vtml-itemname>[NAME]</vtml-itemname>   

             <vtml-itemcover>[COVER]</vtml-itemcover>

             <vtml-itemurl>[URL]</vtml-itemurl>

             <vtml-itemdescription>[DESCRIPTION]</vtml-itemdescription>

             <vtml-itemcategory>[CATEGORY]</vtml-itemcategory>

       </vtml-listitem>

</vtml-group>

Example:

<?xml version="1.0" encoding="UTF-8"?>

<vtml-group>

       <vtml-listitem>

             <vtml-itemname>Hello World</vtml-itemname>   

             <vtml-itemcover>http://www.example.com/media/cover.png</vtml-itemcover>

             <vtml-itemurl>http://www.example.com/media/list.vml1</vtml-itemurl>

             <vtml-itemdescription>This is a list sample</vtml-itemdescription>

             <vtml-itemcategory>Samples</vtml-itemcategory>

       </vtml-listitem>

       <vtml-listitem>

             <vtml-itemname>Hello World 2</vtml-itemname>   

             <vtml-itemcover>http://www.example.com/media/cover.png</vtml-itemcover>

             <vtml-itemurl>http://www.example.com/media/list2.vml1</vtml-itemurl>

             <vtml-itemdescription>This is another list sample</vtml-itemdescription>

             <vtml-itemcategory>Samples</vtml-itemcategory>

       </vtml-listitem>

</vtml-group>   

 

 

Custom list icon definition

The VideoTime client allows each list or group of lists to be defined a custom list icon that will be displayed in the VideoTime client's list manager, these icons are PNG images with the name and file extension of the list followed by .vtmlicon.png

Custom list icon example:

List file name: listname.vml1

List icon file name: listname.vml1.vtmlicon.png

 

For the custom list icon to be displayed on the VideoTime client, the image file must be in the same server location as the list or group of lists associated with the icon.
